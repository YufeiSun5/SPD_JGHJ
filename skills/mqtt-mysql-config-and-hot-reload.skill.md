# [技能] MQTT、多网关、MySQL 与热重载排查流程

## 适用场景

当实时数据不更新、网关未连接、配置改了不生效或任务未重载时，使用本流程。

## 检查顺序

1. 先看 config.env 中数据库连接是否正确。
2. 再查数据库中的网关配置、任务配置、版本配置表。
3. 再看 desktop/main.go 启动日志中数据库初始化、网关加载、热重载是否正常。
4. 最后再查 MQTT Broker 连通性与主题配置。

## 默认事实

1. 多网关真实配置来源是数据库表，不是 config.env 的旧 MQTT 字段。
2. 热重载会在启动后周期性检查配置版本变化。
3. 若数据库未初始化完整，桌面可能进入离线模式或部分页面不可用。

## 复查建议

1. 执行相关 SQL 后，记录可回放的验证步骤。
2. 改完网关配置后，观察日志里是否出现网关新增、删除或重载提示。
3. 改完任务配置后，观察任务加载数量和事件处理是否恢复。
