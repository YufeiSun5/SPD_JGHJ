# 任务管理功能说明 📋

## 功能概述

新增了**任务管理**页面，用于管理系统中的定时任务、数据改变任务和条件事件任务。

---

## 访问路径

**菜单位置**: 系统维护 → 任务管理

**路由**: `#/tasks`

---

## 功能特性

### 1️⃣ 任务列表

- ✅ 显示所有任务配置
- ✅ 支持按任务类型筛选（定时/数据改变/条件事件）
- ✅ 支持按状态筛选（启用/禁用）
- ✅ 支持搜索（任务名称、描述）
- ✅ 实时显示任务状态和更新时间

### 2️⃣ 任务操作

- ✅ **新增任务**: 创建新的任务配置
- ✅ **编辑任务**: 修改现有任务配置
- ✅ **删除任务**: 删除不需要的任务
- ✅ **启用/禁用**: 快速切换任务状态
- ✅ **刷新**: 重新加载任务列表

### 3️⃣ 任务类型支持

#### 📅 定时任务 (TaskType=1)
- 间隔时间（秒）
- Cron表达式（TODO）

#### 🔄 数据改变任务 (TaskType=2)
- 触发变量ID和名称
- 变化类型：
  - `ANY`: 任意变化
  - `INCREASE`: 增加
  - `DECREASE`: 减少
  - `THRESHOLD`: 阈值
  - `FALSE_TO_TRUE`: 0→1
  - `TRUE_TO_FALSE`: 1→0

#### ⚡ 条件事件任务 (TaskType=3)
- 条件表达式（如：temp>50 AND pressure>100）

### 4️⃣ 动作类型支持

1. **HTTP请求** (1): GET/POST/PUT
2. **MQTT发布** (2): 发布消息到MQTT主题
3. **数据库操作** (3): 执行SQL或预定义操作 ⭐
4. **执行脚本** (4): bash/python/powershell
5. **写日志** (5): INFO/WARN/ERROR

## 预定义数据库操作

在 `action_config` 中使用 `operation` 字段指定预定义操作：

### 1. `update_device_status` - 更新设备状态

```json
{
  "operation": "update_device_status",
  "op_params": {
    "device_id": 1,
    "status": 1,
    "remark": "开机运行"
  }
}
```

### 2. `end_device_status` - 结束设备状态

```json
{
  "operation": "end_device_status",
  "op_params": {
    "device_id": 1,
    "remark": "停机"
  }
}
```

### 3. `increment_production_qty` - 增加产量

```json
{
  "operation": "increment_production_qty",
  "op_params": {
    "device_id": 1,
    "ok_qty_delta": 1,
    "ng_qty_delta": 0
  }
}
```

### 4. `log_system_alarm` - 记录系统报警 ⭐

```json
{
  "operation": "log_system_alarm",
  "op_params": {
    "var_id": 88,
    "var_name": "#1焊机错误信息",
    "error_code": "{{new_value}}"
  }
}
```

## 模板变量

在 `action_config` 中可以使用模板变量，系统会自动替换：

| 变量 | 说明 | 示例 |
|------|------|------|
| `{{var_id}}` | 触发变量ID | 88 |
| `{{old_value}}` | 旧值 | 0 |
| `{{new_value}}` | 新值 | 1001 |
| `{{change}}` | 变化量 | 1 |
| `{{trigger_time}}` | 触发时间 | 2025-12-29 10:30:00 |

## 使用建议

### ✅ 适合场景
- 开发和调试任务配置
- 快速创建和测试任务
- 查看任务执行状态
- 临时禁用/启用任务

### ⚠️ 注意事项
- 这是开发者工具，界面简洁实用
- 修改任务后需要触发配置热重载（或重启系统）
- 删除任务前请确认没有依赖
- JSON配置格式错误会导致保存失败
