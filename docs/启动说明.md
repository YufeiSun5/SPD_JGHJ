# IIoT 网关系统 - 启动说明

## 📋 配置步骤

### 1. 修改数据库配置

编辑 `config.env` 文件,设置您的数据库连接信息:

```env
DB_HOST=localhost
DB_PORT=3306           # MySQL用3306, PostgreSQL用5432
DB_USER=root
DB_PASSWORD=your_password_here
DB_NAME=iot_gateway
```

### 2. 导入数据库表结构

使用您提供的建表SQL创建以下表:
- `sys_gateways` - 网关配置表
- `sys_devices` - 设备表
- `sys_variables` - 变量配置表
- `sys_data_history` - 历史数据表
- `sys_alarm_records` - 报警记录表
- `sys_config_version` - 配置版本表

### 3. 配置网关连接

在 `sys_gateways` 表中插入网关配置:

```sql
INSERT INTO sys_gateways (gw_name, status, mqtt_broker, mqtt_client_id, mqtt_user, mqtt_pass, mqtt_topic)
VALUES 
('1号车间网关', 1, 'tcp://192.168.1.100:1883', 'gateway_01', 'admin', 'password', 'factory/line1/#'),
('2号车间网关', 1, 'tcp://192.168.1.101:1883', 'gateway_02', 'admin', 'password', 'factory/line2/#');
```

### 4. 配置测点变量

在 `sys_devices` 和 `sys_variables` 表中配置设备和测点。

产量累计计数器建议额外配置采集层防抖字段：

```sql
UPDATE sys_variables
SET suspicious_value = 0,
    debounce_threshold = 5,
    startup_snapshot_enable = NULL
WHERE var_name LIKE '%产量%';
```

- `suspicious_value = NULL`：默认不开启防抖。
- `debounce_threshold = 5`：计数值大于 5 后才拦截可疑值，保留 `1,0,1` 这类真实小数值复位。
- `startup_snapshot_enable = NULL`：保持兼容旧行为，变化存储/混合存储默认开启启动首帧快照。

详细规则见 `docs/采集层防抖与启动快照说明.md`。

## 🚀 启动系统

### 方式1: 使用环境变量文件

```bash
# Windows PowerShell
$env:DB_HOST="localhost"; $env:DB_PORT="3306"; $env:DB_USER="root"; $env:DB_PASSWORD="123456"; $env:DB_NAME="iot_gateway"; .\iot-gateway.exe

# Linux/Mac
export $(cat config.env | xargs) && ./iot-gateway
```

### 方式2: 直接运行(使用默认配置)

```bash
go run main.go
```

## 📊 数据库读取策略

### **启动时读取一次 (缓存到内存)**

系统启动时会执行以下操作:

1. **加载网关配置** (读取 `sys_gateways` 表)
   - 只在启动时读取一次
   - 加载所有 `status=1` 的网关
   - 每个网关启动独立的 MQTT 连接和 goroutine

2. **加载测点配置** (读取 `sys_variables` 表)
   - 只在启动时读取一次
   - 所有测点配置加载到内存 `map[string]*Tag`
   - 使用 `sync.RWMutex` 保证并发安全
   - 开启防抖的点位会预热 `sys_data_history` 最近非零值，用于判断启动首帧可疑值

3. **补齐采集增强列**
   - 应用启动会幂等补齐 `sys_variables.suspicious_value`
   - 应用启动会幂等补齐 `sys_variables.debounce_threshold`
   - 应用启动会幂等补齐 `sys_variables.startup_snapshot_enable`

### **运行时写入数据库 (高频)**

- **历史数据写入**: ChangeWorker 和 CycleWorker 持续写入 `sys_data_history`
- **报警记录写入**: AlarmWorker 写入 `sys_alarm_records`

### **配置更新机制**

当前实现: 配置保存后更新 `sys_config_version`，后端热重载协程会自动重载测点和网关配置。

注意：采集层防抖的启动历史最近非零值在测点加载/热重载时预热；如果刚调整了产量点位防抖参数，建议观察一轮真实采集确认历史基准符合预期。

## 🔄 多网关扩展

### 添加新网关的步骤:

1. 在 `sys_gateways` 表插入新网关配置
2. 在 `sys_devices` 表添加该网关下的设备
3. 在 `sys_variables` 表配置设备的测点
4. **重启系统** 或调用热加载API (待实现)

每个网关运行在**独立的 goroutine**,互不干扰!

## ⚡ 性能特点

- **内存优先**: 测点配置和实时值全部在内存,读取零延迟
- **读一次**: 启动时加载配置到内存,后续零数据库查询
- **批量写入**: 历史数据和报警记录批量写入,减少IO压力
- **不读 Redis**: 使用 Go 原生 Channel 和 Map,无外部依赖

```bash
cd desktop
wails dev
```

- ✅ 实时显示所有测点数据 (每秒刷新)
- ✅ 显示 Channel 队列状态
- ✅ 报警状态高亮显示
- ✅ 直接从 Go 内存读取,零延迟
