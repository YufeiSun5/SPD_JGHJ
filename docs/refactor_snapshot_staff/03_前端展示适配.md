# 03 — 前端展示适配

> 涉及文件：`desktop/frontend/src/views/ShiftReport.vue`  
> 改动范围：人员列展示 + CSV 导出

---

## 1. 人员列展示改造

### 当前展示方式

```html
<td class="col-staff" :title="fmtStaffFull(s.staff_snapshot)">
  {{ fmtStaff(s.staff_snapshot) }}
</td>
```

当前 `fmtStaff` 只是从 JSON 里提取 `name` 做逗号拼接，没有工时信息。

### 改造后展示方式

**思路**：当 `staff_snapshot` 中的条目包含 `work_sec` 字段时，显示带工时占比的标签；否则降级为原始展示（兼容旧快照数据）。

```javascript
// 改造后 fmtStaff — 展示人员名+工时占比
const fmtStaff = (json) => {
  if (!json) return '-'
  try {
    const arr = JSON.parse(json)
    if (!arr.length) return '-'
    
    // 新格式：含 work_sec 字段
    if (arr[0].work_sec !== undefined) {
      return arr.map(s => {
        const h = Math.floor(s.work_sec / 3600)
        const m = Math.floor((s.work_sec % 3600) / 60)
        const timeStr = h > 0 ? `${h}h${String(m).padStart(2,'0')}m` : `${m}m`
        return `${s.name}(${s.pct}%/${timeStr})`
      }).join('、')
    }
    
    // 旧格式降级：只显示人名
    const names = arr.map(s => s.name)
    return names.length > 3
      ? names.slice(0, 3).join('、') + `…共${names.length}人`
      : names.join('、')
  } catch { return '-' }
}
```

**Tooltip (hover 详情)**：

```javascript
const fmtStaffFull = (json) => {
  if (!json) return ''
  try {
    const arr = JSON.parse(json)
    if (arr[0]?.work_sec !== undefined) {
      return arr.map(s => {
        const h = Math.floor(s.work_sec / 3600)
        const m = Math.floor((s.work_sec % 3600) / 60)
        const timeStr = h > 0 ? `${h}h${String(m).padStart(2,'0')}m` : `${m}m`
        const team = s.team_name ? `[${s.team_name}]` : ''
        return `${s.name}(${s.code || ''}) ${team} ${timeStr} (${s.pct}%)`
      }).join('  |  ')
    }
    return arr.map(s => `${s.name}(${s.code || ''})`).join('  ')
  } catch { return '' }
}
```

### 展示效果（示例）

**表格列**：
```
张三(6.3%/30m)、李四(81.3%/6h30m)、王五(12.5%/1h00m)
```

**Hover Tooltip**：
```
张三(Z003) [A组] 30m (6.3%)  |  李四(L004) [B组] 6h30m (81.3%)  |  王五(W005) [C组] 1h00m (12.5%)
```

## 2. 班组列展示（可选优化）

当前班组列只显示 `team_name`（改造后取工时最大的主班组）。

如果需要显示"多班组"，可以从 `sessions_snapshot` 中提取独立班组名称：

```javascript
const fmtTeam = (snapshot) => {
  // 优先从 sessions_snapshot 取多班组信息
  if (snapshot.sessions_snapshot) {
    try {
      const sessions = JSON.parse(snapshot.sessions_snapshot)
      if (sessions.length > 1) {
        const teams = [...new Set(sessions.map(s => s.team_name))].filter(Boolean)
        return teams.join('/')
      }
    } catch {}
  }
  return snapshot.team_name || '-'
}
```

**展示效果**：`A组/B组/C组`（多班组时）或 `B组`（单班组时）

> **建议**：先不改班组列，因为 `team_name` 已代表"主班组"含义。多班组信息在人员列的工时占比中已经体现。如果用户觉得有必要再加。

## 3. CSV 导出适配

当前 CSV 导出的人员列使用 `fmtStaffFull`：

```javascript
// ShiftReport.vue 第 332 行
fmtStaffFull(s.staff_snapshot)
```

改造后 `fmtStaffFull` 已经自动包含工时信息，CSV 导出无需额外修改。

但为了 CSV 可读性更好，可以专门给 CSV 做一个格式化函数：

```javascript
const fmtStaffCSV = (json) => {
  if (!json) return ''
  try {
    const arr = JSON.parse(json)
    if (arr[0]?.work_sec !== undefined) {
      return arr.map(s => {
        const min = Math.round(s.work_sec / 60)
        const team = s.team_name ? `[${s.team_name}]` : ''
        return `${s.name}${team} ${min}min(${s.pct}%)`
      }).join('; ')
    }
    return arr.map(s => s.name).join('; ')
  } catch { return '' }
}
```

## 4. 人员筛选查询（无需修改）

当前的 `GetShiftSnapshots` 用 `staff_snapshot LIKE '%name%'` 做模糊匹配：

```go
// app_snapshot.go 第 442 行
query = query.Where("staff_snapshot LIKE ?", "%"+staffName+"%")
```

改造后 `staff_snapshot` 仍然包含 `name` 字段，LIKE 模糊匹配依然有效，**无需修改查询逻辑**。
