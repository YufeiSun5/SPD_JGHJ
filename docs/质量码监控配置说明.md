# 质量码监控配置说明

## 📌 **功能概述**

**质量码 = 连接状态 = 设备在线/离线状态**

- **质量码 192 (0xC0)** → 转换为 `1` → **设备在线，通信正常** ✅
- **质量码 0 或其他非192值** → 转换为 `0` → **设备离线，通信中断** ❌

---

## 🎯 **应用场景**

### **场景1：设备上线自动更新状态**

```
问题：设备从离线恢复在线时，需要自动更新设备状态为"运行"
解决：使用 QUALITY_GOOD 监控，当质量码从 0→1 时触发
```

**示例**：
```
时间线：
10:00 - 设备离线，质量码=0，#2开机状态=0（不可信）
10:05 - 设备恢复在线，质量码=192→1，#2开机状态=1
        ↓
        触发任务：更新设备状态为"运行"
```

### **场景2：设备离线告警**

```
问题：设备离线时需要发送告警通知
解决：使用 QUALITY_BAD 监控，当质量码从 1→0 时触发
```

## 📊 **质量码映射规则**

| MQTT 质量码 | 内部 Quality | 含义 | 说明 |
|------------|-------------|------|------|
| 192 (0xC0) | 1 | Good | 设备在线，数据有效 ✅ |
| 0 或其他值  | 0 | Bad  | 设备离线，数据无效 ❌ |

**代码实现位置**：`workers/logic_worker.go`

```go
// 解析质量码 (属性3)
quality := 0
if rawQuality, ok := obj["3"].(float64); ok {
    if int(rawQuality) == 192 {
        quality = 1 // Good
    } else {
        quality = 0 // Bad
    }
}
```

## ⚠️ **注意事项**

### **1. 首次启动不触发**

系统启动后第一次收到数据时，`IsFirstUpdate=true`，不会触发任务。这是为了避免冷启动时的误触发。

### **2. 质量码变化才触发**

只有质量码真正发生变化时才会触发：
- `QUALITY_GOOD`: 必须从 `0→1`
- `QUALITY_BAD`: 必须从 `1→0`

如果质量码一直是 `1`，即使数据值变化，也不会触发质量码任务。

### **3. 与数据变化任务独立**

质量码监控和数据变化监控是独立的：
- 数据值从 `0→1` 可以触发 `FALSE_TO_TRUE` 任务
- 质量码从 `0→1` 可以触发 `QUALITY_GOOD` 任务
- 两者可以同时触发，互不影响

### **4. 性能考虑**

每次数据更新都会检查质量码变化，但只有真正变化时才会触发任务，性能开销极小。
